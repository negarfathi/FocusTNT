# FocusTNT

FocusTNT is a lightweight, tool-agnostic preprocessing framework for the termination and non-termination analysis of C/C++ programs that reshapes the verification task before existing analyzers are executed.

* **Loop-based slicing** – isolates code relevant to a target loop
* **Input-driven concretization** – replaces nondeterministic inputs with concrete values derived from test cases

These transformations allow analyzers to focus on the most relevant parts of the program, often improving analysis effectiveness.

FocusTNT evaluates four configurations:

* **base** – no preprocessing
* **slice** – loop-based slicing only
* **cncrt** – input-driven concretization only
* **slice+cncrt** – slicing followed by concretization

The framework is evaluated using several state-of-the-art verification tools, including:
[Athena](https://github.com/negarfathi/Athena),
[Proton](https://github.com/kumarmadhukar/term/tree/main/proton),
[UAutomizer](https://www.ultimate-pa.org/automizer/),
[AProVE](https://aprove.informatik.rwth-aachen.de/),
[CPAchecker](https://cpachecker.sosy-lab.org/), and
[2LS](https://github.com/diffblue/2LS) on the benchmark suite produced by [Shi et al.](https://github.com/FSE2022benchmarks/-FSE-2022-Termination/tree/v1.0).



## Requirements

Before installing FocusTNT, ensure that the following dependencies are installed:

* C/C++ compiler with C++23 support (e.g., clang or gcc)
* LLVM/Clang (tested with LLVM 21)
* CMake (version ≥ 3.27)
* Make
* Git
* Docker
* Curl
* Unzip



## Installation

Clone the repository:

```bash
git clone https://github.com/negarfathi/FocusTNT.git
```

Run the installation script from the repository root:

```bash
./install_FocusTNT.sh
```

This script installs all required dependencies, including:

* [Frama-C](https://frama-c.com) (for loop-based slicing)
* [xlnt](https://github.com/xlnt-community/xlnt) (for exporting results to Excel)
* Athena, Proton, UAutomizer, AProVE, CPAchecker, and 2LS (state-of-the-art verification tools used for evaluation)



## Execution

Run the following script from the repository root:

```bash
./run_FocusTNT.sh
```

The script executes FocusTNT across all benchmarks, analyzers, and preprocessing configurations. It runs the following command:

```
<path/to/FocusTNT> <path/to/SourceCode.c> <path/to/TestCase.csv> \
    --tool=<tool-name> \
    --benchmark=<benchmark-name> \
    --slicer=<true-or-false> \
    --concretizer=<true-or-false> \
    --timeout=<timeout>
```

Where:

* `<path/to/FocusTNT>` is the path to the FocusTNT executable.
* `<path/to/SourceCode.c>` is the path to the C source file to be analyzed.
* `<path/to/TestCase.csv>` is the path to the CSV file containing the test cases used for input concretization.
* `<tool-name>` specifies the backend verification tool (`Athena`, `PROTON`, `UAutomizer`, `AProVE`, `CPAchecker`, `2LS`).
* `<benchmark-name>` specifies the benchmark set. It should be set to `FSE` for the benchmark suite of Shi et al.
* `--slicer` and `--concretizer` can be `true` or `false` to enable or disable the respective preprocessing steps.
* `--timeout` specifies the maximum runtime (in seconds) for preprocessing and analysis.

The configuration flags correspond to the following modes:

| Configuration   | Flags                                | Description                        |
| --------------- | ------------------------------------ | ---------------------------------- |
| **base**        | `--slicer=false --concretizer=false` | No preprocessing                   |
| **slice**       | `--slicer=true --concretizer=false`  | Loop-based slicing                 |
| **cncrt**       | `--slicer=false --concretizer=true`  | Input-driven concretization        |
| **slice+cncrt** | `--slicer=true --concretizer=true`   | Slicing followed by concretization |



## Output

After running the experiments, FocusTNT produces two types of output.

#### 1. Experiment Result Directories

All reproduced experiment results are written to the following directory, relative to the repository root:

```
ReproducedResults/
```

Results are organized by verification tool and preprocessing configuration:

```
ReproducedResults/
	<tool-name>/
		base/
		slice/
		cncrt/
		slice+cncrt/
```

Where:

* `<tool-name>` is one of the evaluated analyzers
  (`Athena`, `PROTON`, `UAutomizer`, `AProVE`, `CPAchecker`, `2LS`)

Each directory contains the detailed outputs generated by the corresponding
verification tool under the specified preprocessing configuration.



#### 2. Excel Summary of Evaluation Results

FocusTNT also generates an Excel file summarizing the evaluation results
across all tools and configurations.

The file is created in the repository root:

```
<benchmark-name> - Evaluation Results.xlsx
```

This file aggregates the results from all experiments and provides a tabular summary suitable for further analysis.
